기본 github-flow 사용법

1. git pull
2. git branch <브랜치명>
3. git checkout <브랜치명>
4. 코드 작성
5. git add <파일명>
6. git commit -m "원하는 메시지"
7. 깃헙에서 PR 후 리뷰
8. 문제가 없다면 merge

기존 코드와 별개로 새로 만들거나 그렇게 만들어진 파일을 수정한 경우, 
그 파일만 별도로 add 한 후, sub 브랜치에도 push 해주세요.

**

기존 코드 수정은 최소화 할것
커밋 메시지는 알아보기 쉽게 작성할것
기존 코드에 수정할 점이 있다면 주석처리 할것!!
